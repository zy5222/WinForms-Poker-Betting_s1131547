# 五張撲克牌投注系統_1131547

##  核心功能介紹
* **完整投注與動態扣款系統**：
  打破傳統單純發牌的限制，加入真實賭場的資金邏輯。預設提供 1,000,000 元初始資金，玩家可自由輸入押注金額。系統採用「先扣款，後發獎」的機制，確認下注後立即從總資金扣除，並即時更新於介面上。
* **精準牌型判斷與賠率結算**：
  內建九大經典撲克牌型判斷邏輯（一對、兩對、三條、順子、同花、葫蘆、四條、同花順、皇家同花順）。開牌結算時，系統會自動比對賠率表（最高賠率達 250 倍），並精準計算該局贏得獎金，加總回玩家的總資金池中。
* **換牌機制實作**：
  完美模擬撲克規則，發牌後玩家可針對不需要的卡片點擊「牌背」進行翻牌標記，點擊「換牌」後系統會自未發出的牌堆中依序抽取新牌替換，且限制每局僅能進行一次換牌。

##  防呆與錯誤攔截機制

**1. 四階段輸入檢查：**
在玩家點擊「押注」的瞬間，系統會依序進行以下嚴格檢查：
* **空值攔截**：檢查輸入框是否留白或僅包含空白字元 (`IsNullOrWhiteSpace`)。
* **型態驗證**：捨棄易當機的 `Parse`，改用安全的 `long.TryParse`，攔截英文字母、特殊符號。
* **邏輯防呆**：阻擋輸入「0」或「負數」的下注。
* **破產檢測**：精準比對「押注金額」與「目前總資金」，若餘額不足將即時跳出警告視窗並中斷扣款流程。

**2. 遊戲狀態機與死鎖防護：**
利用控制項的 `Enabled` 與 `ReadOnly` 屬性，建立強勢的單向遊戲流程，防止玩家誤觸：
* **階段一 (等待下注)**：鎖定所有發牌與結算按鈕，強迫「未下注不可發牌」。
* **階段二 (發牌與換牌)**：下注成功後，鎖定下注金額輸入框防止中途偷改賭金。
* **階段三 (結算與重置)**：開牌結算後，徹底鎖定桌上所有卡片的點擊事件 (`Click`) 與換牌按鈕，避免玩家在跨回合期間翻動卡片產生靈異現象，並自動清空前次下注額，無縫銜接下一局。

##  格外功能 

**1. 突破型態極限：整數溢位防護**
因為高階牌型（如皇家同花順）賠率高達 250 倍，若玩家進行千萬等級的梭哈下注（例如：50,000,000 * 250 = 125億），將輕易突破 C# 預設 32 位元 `int` 變數的最大極限，導致Overflow讓獎金反轉成負數。
所以我將所有與資金計算相關的變數（`totalMoney`, `currentBet`, `winnings`）全部重構並升級為 64 位元的 **`long`** 型態，確保龐大數值計算的正確性。

**2. 開發者專屬測試環境 (Cheat Keys System)**
為了大幅降低測試特定牌型（機率極低）的時間成本，因此設計了的鍵盤事件 (`KeyPress`)：
* **一鍵配牌**：在發牌前的準備階段，開發者按下特定字母鍵（如 `q`=同花大順, `r`=鐵支, `t`=葫蘆），系統會直接竄改 `playerPoker` 陣列的資料，瞬間配出指定神牌。
* **干擾防護罩**：為避免玩家在輸入下注金額時，因鍵盤敲擊不慎觸發翻牌事件，特別加入了 `isCheatKey` 旗標與階段判斷 (`btnDealCard.Enabled == false`)，將「正常輸入」與「作弊指令」完美隔離。
